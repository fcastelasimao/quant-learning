## 🔴 Pre-Paper-Trading (complete before using real or paper money)

These items represent the gap between a research backtest and a live strategy.
Do not skip them -- each one addresses a known distortion in the current results.

- [ ] **Switch to total return prices (dividends + interest)**
  Currently the backtest uses price returns only. This materially understates
  performance, particularly for bond-heavy allocations.
  - TLT at 3% average coupon yield over 20 years adds ~80% cumulative return
  - SHY currently yielding ~4-5% -- enormous drag to ignore
  - GLD pays no dividend -- correct to exclude, no change needed
  Fix: change `data.py` to pass `auto_adjust=True` to yfinance. One line.
  After implementing, re-run the full 2006-2026 backtest to establish a new
  corrected baseline. All previous results will be slightly understated.
  Verify a spot-check: TLT total return vs price return for a known period.

- [ ] **Add transaction costs to backtest**
  Every monthly rebalancing trade is currently assumed free. Real costs:
  - Brokerage commission: ~£0-5 per trade (most modern brokers free)
  - Bid-ask spread: ~0.01-0.05% on liquid ETFs like SPY, TLT, GLD
  - Stamp duty: 0.5% on UK equity purchases (ETFs listed in London)
  - FX conversion: ~0.5-1.5% if buying USD-denominated ETFs from a GBP account
  Add `TRANSACTION_COST_PCT = 0.001` to `config.py` (0.1% per round trip is
  conservative but realistic for a UK retail investor). Subtract from portfolio
  value on each rebalancing step in `run_backtest()` in `backtest.py`.
  Key question: does the strategy still beat 60/40 after costs on small portfolios?

- [ ] **Model tax drag**
  Monthly rebalancing is a tax event in most jurisdictions. Each rebalancing
  trade that realises a gain is taxable:
  - UK: Capital Gains Tax at 18-24% above the £3,000 annual allowance (2026)
  - Tax-sheltered wrappers (ISA, SIPP) eliminate this entirely -- consider
    noting that this strategy is best implemented inside an ISA or SIPP
  Add a note to the README and config.py clarifying that results assume a
  tax-sheltered account. For taxable accounts, model a conservative 15% drag
  on realised gains per year.

- [ ] **Test with total return prices then re-run the original Dalio 5-ETF allocation**
  Dalio's original weights as the purest baseline:
  ```python
  TARGET_ALLOCATION = {
      "SPY": 0.30,   # Stocks
      "TLT": 0.40,   # Long-term bonds
      "IEF": 0.15,   # Intermediate bonds
      "GLD": 0.075,  # Gold
      "GSG": 0.075,  # Commodities
  }
  ```
  Run `RUN_MODE = "full_backtest"` on 2006-2026 with total return prices.
  Key question: how much does 20 years of engineering actually add over
  simply buying the original Dalio allocation unchanged?

- [ ] **Add slippage model**
  Market orders fill at worse than the last price, especially for less liquid
  instruments. Add `SLIPPAGE_PCT = 0.0005` (0.05% per trade) to `config.py`.
  For GSG and IWD which are less liquid than SPY/TLT, consider higher slippage.
  Combine with transaction costs into a single `_apply_trade_costs()` helper
  in `backtest.py` so both can be toggled independently.

- [ ] **Stress test with historical scenario analysis**
  The backtest captures 2008 and 2022 but not all relevant scenarios.
  Add a `RUN_MODE = "scenario"` that slices the backtest to specific periods
  and reports metrics for each:
  - 2008-2009 financial crisis (Sep 2008 - Mar 2009)
  - 2010-2012 European debt crisis
  - 2013 Taper Tantrum (bond-only shock)
  - 2018 Q4 equity correction
  - 2020 Covid crash (Feb-Mar 2020)
  - 2022 rate shock (Jan-Dec 2022)
  This gives a richer picture of drawdown behaviour than max drawdown alone
  and is the kind of analysis a professional would present to a client.

- [ ] **Add fractional share support to rebalancing instructions**
  Currently rebalancing shows dollar amounts but not share counts, which
  is what a broker requires. Add share count calculation to `portfolio.py`:
  ```python
  shares_to_trade = dollar_amount / current_price
  ```
  For small portfolios (<£5,000) some trades may be <1 share -- flag these
  as "fractional share required" and note whether the user's broker supports it.
  Interactive Brokers supports fractional shares; many UK brokers do not.

- [ ] **Minimum portfolio size analysis**
  At £10,000 with 8 assets, each position is ~£1,000-2,500. With a 5%
  rebalancing threshold and transaction costs, determine the minimum portfolio
  size below which rebalancing costs erode returns to the point where
  buy-and-hold is strictly superior. Document this clearly in the README.

## 🟢 Rebalancing Methodology

- [ ] **Consider rebalancing timing to reduce timing luck**
  Three options in increasing complexity:

  1. **Month-start (`"MS"`)** -- change `"ME"` to `"MS"` in `config.py`.
     Avoids month-end window dressing. One-line change.

  2. **Mid-month (15th)** -- resample on the 15th of each month.
     Avoids both turn-of-month and month-end distortions.
     Requires a small custom resampling function in `backtest.py`.

  3. **Overlapping portfolios (Newfound Research method)** -- split into
     4 sub-portfolios rebalancing on different weeks. Eliminates timing
     luck while keeping total turnover identical. Significant effort.

  Low priority given highly liquid ETFs. Worth revisiting before publishing.

- [ ] **Threshold rebalancing vs calendar rebalancing comparison**
  Currently: rebalance every month regardless of drift.
  Alternative: rebalance only when an asset drifts beyond threshold.
  Test: run backtest with `REBALANCE_THRESHOLD = 0.10` (10% drift trigger)
  vs current monthly rebalancing. Expect: fewer trades, lower costs, slightly
  higher drawdown. May be better for taxable accounts.

  - TLT → IDTL (iShares $ Treasury Bond 20+yr, London-listed)
  - GLD → IGLN (iShares Physical Gold, London-listed)
  Run a spot-check backtest comparing US vs UK-listed versions on a
  common period to confirm tracking error is acceptable.

- [ ] **Define paper trading protocol**
  - Run `RUN_MODE = "oos_evaluate"` at month-end to get rebalancing instructions
  - Execute trades in paper account within 2 trading days of month-end
  - Log actual fill prices vs model prices to measure real-world slippage
  - Track paper account performance against the model for 3-6 months
  - If paper vs model divergence is consistently <0.5% per month, proceed
    to live trading

- [ ] **Define go/no-go criteria for live trading**
  Do not move to live money without clear criteria:
  - Paper trading period: minimum 3 months (6 preferred)
  - Paper vs model tracking error: <1% cumulative over the period
  - Maximum paper drawdown not exceeded vs model prediction: <2% gap
  - Personal conviction in the macro thesis still intact
  - Tax wrapper confirmed: ISA or SIPP preferred to avoid CGT on rebalancing
